# Quickstart

This guide will walk you through creating your first world, defining entities, executing events, and seeing AI-driven state changes.

<Callout type="info" title="Prerequisites">
  Set the `URL` to the API endpoint (`https://api.narrativeprotocol.com`)
</Callout>

## Getting an API key

Before making any API calls, you need to authenticate with the system. Register at [https://studio.narrativeprotocol.com](https://studio.narrativeprotocol.com) to get API key that will be used for all subsequent requests.

---

## Blueprint Layer

The Blueprint layer is where you define your world's structure. This includes creating worlds, entity schemas with attributes, and events with versioned behavior. These definitions are separate from the runtime execution.

### Create Your World

A World is the top-level container for your simulation. It holds `entities`, `events`, and can have multiple `deployments`. The `promptSeed` helps the AI understand the overall context of your world.

### Define an Entity Schema

An Entity Schema defines the structure for entities in your world. For example, a "horse" schema defines what attributes (name, speed, wins) all horses will have. You can define attributes during entity schema creation, including `type`, `constraints`, and `defaultValue`.

Attribute `type` values:

- `string`
- `integer`
- `float`
- `boolean`

Attribute `constraints` fields:

- `min` (number)
- `max` (number)
- `enum` (string array)

### Create an Event

Events define what can happen in your world.

`firstVersion` fields:

- `inputSchema`: Defines the input payload expected when executing the event. Input is expected during event execution.
- `readEntities`: Lists entity schemas the event can read.
- `entityChangeSettings`: Per-schema mutation mode (`partial`, `full`, `append`). `partial` will merge into existing values, `full` will replace existing values, `append` means appending the values to as array.
- `outputSchema`: Defines the execution result, you can declare any field here and the AI will fill-in.
- `behaviorPrompt`: prompt used by AI during execution mode.
- `mutationSettings`: Execution mode config (`ai` or `direct`; `direct` uses a template).
- `executionSettings`: Execution visibility and triggers (`admin`, `public`, `cron`) with optional `priceUsd` and `cron` config.

For more details visit [events](/docs/concepts/events) page.

## Live Layer

The Live layer is where your simulation actually runs. Deployments are isolated runtime instances of a world.

### Create a Deployment

A Deployment is a running instance of a world with its own isolated state.

The `targetChains` option determines where event data is stored on-chain:

- `[]` - No on-chain storage (default)
- `["solana-devnet"]` - Push to Solana devnet
- `["near-testnet"]` - Push to NEAR testnet
- `["solana-devnet", "near-testnet"]` - Push to both chains
