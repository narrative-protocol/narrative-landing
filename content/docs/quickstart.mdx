# Quickstart

This guide will walk you through creating your first world, defining entities, executing events, and seeing AI-driven state changes.

<Callout type="info" title="Prerequisites">
  Set the `URL` to the API endpoint (`https://api.narrativeprotocol.com`)
</Callout>

## Getting an API key

Before making any API calls, you need to authenticate with the system. Register at [https://studio.narrativeprotocol.com](https://studio.narrativeprotocol.com) to get API key that will be used for all subsequent requests.

---

## Blueprint Layer

The Blueprint layer is where you define your world's structure. This includes creating worlds, entity schemas with attributes, and events with versioned behavior. These definitions are separate from the runtime execution.

### Create Your First World

A World is the top-level container for your simulation. It holds entity schemas, events, and can have multiple deployments. The promptSeed helps the AI understand the overall context of your world.

**Request:**

```bash
curl -X POST https://api.narrativeprotocol.com/api/worlds \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d '{
    "name": "Horse Racing",
    "description": "A horse racing simulation world",
    "domainTags": ["sports", "simulation"],
    "promptSeed": "This world simulates realistic horse racing events."
  }'
```

**Response:**

```json

  "data": {
```

### Define an Entity Schema

An Entity Schema defines the structure for entities in your world. For example, a "horse" schema defines what attributes (name, speed, wins) all horses will have.

**Request:**

```bash
curl -X POST https://api.narrativeprotocol.com/api/entity-schemas \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d '{
    "worldId": <world-id>,
    "name": "horse",
    "description": "A racing horse"
  }'
```

**Response:**

```json
{
  "success": true,
  "data": {
    "id": 53,
    "worldId": 39,
    "name": "horse",
    "description": "A racing horse",
    "createdAt": "2026-02-18T04:13:49.117Z",
    "updatedAt": "2026-02-18T04:13:49.117Z",
    "attributeDefinitions": []
  }
}
```

### Add Attributes to Entity Schema

Attributes define the specific properties each entity instance can have. You can specify type (string, integer, float, boolean), constraints, and default values.

```bash
# Speed rating attribute
curl -X POST https://api.narrativeprotocol.com/api/attribute-definitions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d '{
    "entitySchemaId": <schema-id>,
    "name": "speed_rating",
    "type": "float",
    "constraints": { "min": 0, "max": 1 },
    "defaultValue": 0.5
  }'
```

### Create an Event

Events define what can happen in your world. Each event has versions that become immutable once published, ensuring reproducibility.

```bash
curl -X POST https://api.narrativeprotocol.com/api/events \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d '{
    "worldId": <world-id>,
    "name": "race_result",
    "description": "Resolves a race and updates horse stats",
    "firstVersion": {
      "inputSchema": { "raceId": "string" },
      "reads": [<schema-id>],
      "stateChangeSchema": { "horse": "partial" },
      "resultSchema": { "winner": "string", "time": "string" },
      "behaviorPrompt": "Determine the race winner based on horse speed ratings. Update the winners wins count."
    }
  }'
```

## Live Layer

The Live layer is where your simulation actually runs. Deployments are isolated runtime instances of a world.

### Create a Deployment

A Deployment is a running instance of a world with its own isolated state.

```bash
curl -X POST https://api.narrativeprotocol.com/api/deployments \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d '{
    "worldId": <world-id>,
    "name": "Season 1",
    "targetChains": ["near-testnet"],
    "firstBinding": {
      "eventId": <event-id>,
      "eventVersion": <event-version>
    }
  }'
```

The `targetChains` option determines where event data is stored on-chain:

- `[]` - No on-chain storage (default)
- `["solana-devnet"]` - Push to Solana devnet
- `["near-testnet"]` - Push to NEAR testnet
- `["solana-devnet", "near-testnet"]` - Push to both chains

### Create Entity Instances

Entity Instances are the actual data in your deployment.

```bash
curl -X POST https://api.narrativeprotocol.com/api/entity-instances \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d '{
    "deploymentId": <deployment-id>,
    "entitySchemaId": <schema-id>,
    "instanceId": "HORSE_1",
    "state": {
      "name": "Midnight Comet",
      "speed_rating": 0.85
    }
  }'
```

### Execute an Event

This is where the magic happens! Executing an event triggers the AI engine.

```bash
curl -X POST https://api.narrativeprotocol.com/api/deployments/<deployment-id>/execute \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer <your-token>" \
  -d '{
    "eventId": <event-id>,
    "input": { "raceId": "RACE_001" }
  }'
```

### View Updated State

After executing an event, you can query the entity instances to see how the state has changed.

```bash
curl https://api.narrativeprotocol.com/api/entity-instances?deploymentId=<deployment-id> \
  -H "Authorization: Bearer <your-token>"
```
